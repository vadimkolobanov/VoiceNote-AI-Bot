# Предложения по улучшению промптов ИИ и UI бота

## 🎯 Промпты для LLM

### 1. **Улучшение промпта для извлечения напоминаний** (`extract_reminder_details`)

**Текущая проблема:**
- Промпт слишком технический и длинный
- Недостаточно примеров для edge cases
- Не учитывает контекст пользователя (например, рабочие дни)

**Предложения:**

```python
# Добавить в промпт:
- Учет рабочих дней (понедельник-пятница) для фраз типа "в рабочий день"
- Обработка относительных фраз: "через неделю", "через месяц"
- Улучшенная обработка времени суток: "рано утром" (7:00), "поздно вечером" (22:00)
- Поддержка праздничных дней и выходных
- Обработка фраз типа "каждый второй вторник"
```

**Конкретные улучшения:**
1. Добавить больше примеров в промпт (особенно для сложных случаев)
2. Улучшить обработку неоднозначных формулировок
3. Добавить валидацию извлеченных данных (проверка на разумность дат)

### 2. **Промпт для генерации дайджеста** (`generate_digest_text`)

**Текущая проблема:**
- Слишком формальный тон
- Недостаточно персонализации
- Не учитывает контекст (например, если много пропущенных задач - более мягкий тон)

**Предложения:**

```python
# Улучшенный промпт должен:
1. Адаптировать тон в зависимости от контекста:
   - Если много задач - мотивирующий, но не давящий
   - Если пропущенные задачи - поддерживающий, не осуждающий
   - Если задач мало - позитивный и вдохновляющий

2. Добавить персонализацию:
   - Учитывать имя пользователя в разных местах
   - Использовать разные варианты приветствий
   - Добавить небольшие мотивационные фразы

3. Улучшить структуру:
   - Более четкое разделение блоков
   - Использование эмодзи для визуального разделения
   - Краткие, но информативные формулировки
```

### 3. **Промпт для извлечения списка покупок** (`extract_shopping_list`)

**Текущая проблема:**
- Недостаточно четкие правила для единиц измерения
- Не учитывает контекст магазина/рынка
- Проблемы с обработкой количества

**Предложения:**

```python
# Улучшения:
1. Более четкие правила для единиц измерения:
   - "молоко" -> "молоко, 1 л" (если не указано)
   - "хлеб" -> "хлеб, 1 шт" (если не указано)
   - Сохранять оригинальные единицы, если указаны

2. Улучшенная обработка контекста:
   - "в Пятерочке" -> добавить в название товара или отдельное поле
   - "на рынке" -> отметить как рынок
   - "в аптеке" -> категория "Аптека"

3. Обработка списков:
   - "молоко, хлеб, яйца" -> правильно разделить на товары
   - "2 кг сахара, 3 литра молока" -> сохранить количество
```

### 4. **Промпт для классификации намерений** (`classify_intent`)

**Текущая проблема:**
- Слишком простая классификация
- Не учитывает контекст предыдущих сообщений
- Может неправильно классифицировать смешанные намерения

**Предложения:**

```python
# Добавить:
1. Более детальную классификацию:
   - CREATE_NOTE_WITH_DATE (заметка с датой)
   - CREATE_NOTE_WITHOUT_DATE (заметка без даты)
   - CREATE_HABIT (привычка)
   - CREATE_BIRTHDAY (день рождения)

2. Обработка смешанных намерений:
   - Если есть и список покупок, и дата -> приоритет дате
   - Если есть и задача, и привычка -> уточнить у пользователя

3. Улучшенная обработка контекста:
   - Учитывать предыдущие сообщения
   - Понимать продолжение диалога
```

## 🎨 UI и тексты бота

### 1. **Главное меню**

**Текущие проблемы:**
- Слишком много информации в одном сообщении
- Недостаточно визуального разделения
- Статистика может быть перегружена

**Предложения:**

```python
# Улучшенный текст главного меню:
text = (
    f"🏠 {hbold('Главное меню')}\n\n"
    f"📊 {hbold(f'Уровень {user_level}')} | {progress_percent}% до следующего\n"
    f"📝 Создано заметок: {total_notes}\n\n"
    f"💬 {hbold('Что я умею:')}\n"
    f"• Создавать заметки из текста или голоса\n"
    f"• Напоминать о важных делах\n"
    f"• Ведить списки покупок\n"
    f"• Отслеживать дни рождения\n\n"
    f"👉 {hitalic('Просто отправьте мне любое сообщение!')}"
)
```

### 2. **Сообщения об ошибках**

**Текущие проблемы:**
- Слишком технические формулировки
- Недостаточно дружелюбный тон
- Не предлагают решения

**Предложения:**

```python

# Использовать:
"😔 Упс, что-то пошло не так при обработке вашей заметки.\n\n"
"Попробуйте отправить сообщение еще раз. Если проблема повторится, "
"напишите нам через меню «❓ Помощь» → «Сообщить о проблеме»."

# Вместо: "❌ Не удалось распознать речь."
# Использовать:
"🎤 К сожалению, я не смог разобрать ваше голосовое сообщение.\n\n"
"💡 {hbold('Советы:')}\n"
"• Говорите четче и ближе к микрофону\n"
"• Проверьте, что запись не слишком тихая\n"
"• Попробуйте отправить текстовое сообщение"
```

### 5. **Напоминания**

**Текущие проблемы:**
- Слишком формальный тон
- Недостаточно контекста
- Не учитывают важность задачи

**Предложения:**

```python
# Более дружелюбные напоминания:
header = "🔔 Напоминание" if is_pre_reminder else "❗️ ВАЖНО: Время выполнить задачу!"

text = (
    f"{header}\n\n"
    f"📝 {hbold('Задача:')}\n"
    f"{hcode(note_text)}\n\n"
    f"⏰ {hbold('Срок:')} {hitalic(formatted_due_date)}\n\n"
    f"💪 {hitalic('Вы справитесь!')}"
)

# Для предварительных напоминаний:
text = (
    f"🔔 {hbold('Напоминание заранее')}\n\n"
    f"Через {pre_reminder_minutes} минут у вас:\n"
    f"📝 {hbold(note_text)}\n\n"
    f"⏰ {hitalic(formatted_due_date)}\n\n"
    f"💡 Подготовьтесь заранее!"
)
```
