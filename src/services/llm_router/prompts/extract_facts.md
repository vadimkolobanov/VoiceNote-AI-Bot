# What counts as a fact

✅ Долгоиграющие сведения:
- Имена, роли, отношения близких («жена — Диана», «сын Миша 4 года», «коллега Аня — дизайнер»)
- Регулярные предпочтения («не ем мясо», «люблю эспрессо», «бегаю по утрам»)
- Места («работаю в Сбере», «живу в Москве, район Сокол», «дача — Истра»)
- Расписания и привычки на постоянной основе («каждый понедельник тренировка», «по средам отдаю Мишу в сад»)
- Профессия / контекст работы

❌ НЕ факты (не возвращай):
- Разовые задачи и напоминания («завтра встреча с Аней» → это момент, не факт)
- Эмоциональные состояния прямо сейчас («устал», «не выспался»)
- Однократные события («вчера был на концерте»)
- Точное время будущих встреч

# Normalization rules

- Имена приводи к именительному падежу: «Диане» → «Диана»
- Если факт уже есть в списке выше — возвращай тот же `key`, обновлённое `value` пойдёт через UPSERT
- `key` — короткий, машинный slug (lowercase, латиница если возможно): `wife`, `son_misha`, `colleague_anya`, `lives_in`, `works_at`, `pref_no_meat`. Для людей — `<role>_<name_lc>`.
- `value` — объект с человекочитаемыми полями. Для person: `{name, role, details?}`. Для preference: `{summary}`. Для place: `{label, address?}`. Для schedule: `{when, what}`.

# Output schema (строгий JSON)

```json
{
  "facts": [
    {
      "kind": "person | place | preference | schedule | other",
      "key": "string, slug",
      "value": { "...": "..." },
      "confidence": 0.0
    }
  ]
}
```

- Если фактов нет — верни `{"facts": []}`. Не выдумывай.
- `confidence`: 0.9+ если в тексте прямо названо, 0.7 если выводимо из контекста, ниже 0.6 — не возвращай.
